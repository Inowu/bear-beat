# Repo Rules (Required)

- Mantén PRs pequeños y enfocados. Si el cambio crece, divídelo en varios PRs.
- Ejecuta tests siempre y deja evidencia en el PR (comandos + resultado).
- Prohibido loguear PII o datos de pago. Ejemplos de PII: email, teléfono, dirección, IP. Ejemplos de pago: payloads completos de Stripe/PayPal/Conekta, last4, tokens, IDs asociados a una persona.

Notas:
- Guías específicas del backend: `backend/AGENTS.md`.
